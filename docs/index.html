Capabilities

Everything you need to understand your workflow

BLT Timer goes beyond raw counts — it surfaces patterns that help you optimise focus, rest, and coding habits.

Real-time Tracking

Counts keystrokes, mouse distance, left/right clicks, and scroll steps every second via a low-overhead X11/XInput2 native library.

Live Visual Dashboard

Smooth per-second bar charts, 60-second rolling history, and real-time stat cards built with custom Flutter CustomPainter.

Daily Summaries

Persistent daily totals auto-saved across sessions. Manual reset button lets you start fresh at any time. Stored locally — never leaves your device by default.

VS Code AI Prompt Tracker

Automatically discovers and counts AI prompts sent in VS Code and VS Code Insiders to help you understand your AI-assisted coding cadence.

Optional API Integration

Push anonymised session summaries to the BLT Fresh backend over HTTPS. Configurable apiUrl, userId, and projectId.

Privacy by Design

Zero keystroke content, zero cursor coordinates, zero window titles. Only aggregate numeric metrics — you stay in full control.

Architecture

How BLT Timer works

A thin native C library handles low-level X11 events. Flutter polls it every 10 ms and aggregates into per-second updates shown in the dashboard.

  1. X11 / XInput2 Events

    The native activity_tracker.c library hooks into X11's XInput2 extension to capture raw device events with minimal overhead. No window IDs or coordinates are stored.

    1
    linux/activity_tracker.c
  2. lib/native_tracker.dart
    2

    FFI Bridge (10 ms poll)

    Dart FFI calls the shared library every 10 ms, pulling aggregated counters — no raw events cross the boundary. Bindings live in native_tracker.dart.

  3. ActivityService (1 s aggregation)

    Per-second averages and cumulative totals are computed and broadcast via a Dart Stream, keeping the UI layer free of business logic.

    3
    lib/activity_service.dart
  4. lib/main.dart
    4

    Flutter UI (CustomPainter charts)

    setState rebuilds at 1 Hz, rendering stat cards and bar charts using custom painters — no heavyweight charting dependency.

  5. Optional: BLT Backend (opt-in)

    When enabled, TrackerIntegration POSTs anonymised session payloads to a Cloudflare Worker endpoint. HTTPS-only, no PII transmitted.

    5
    lib/tracker_integration.dart

Data pipeline

X11 Events → C Library (10 ms) → FFI Poll → ActivityService (1 s) → Flutter UI → [Optional] BLT API

Setup

Get running in minutes

1

Install system dependencies

sudo apt-get update
sudo apt-get install -y libx11-dev libxtst-dev libxi-dev build-essential
2

Set up Flutter (Linux desktop)

git clone https://github.com/flutter/flutter.git -b stable
export PATH="$PATH:$HOME/flutter/bin"
flutter doctor
flutter config --enable-linux-desktop
3

Clone, build native library, and run

git clone https://github.com/OWASP-BLT/BLT-Timer-Flutter.git
cd BLT-Timer-Flutter/linux
./build.sh
cd ..
flutter pub get
flutter run -d linux

Privacy

What we track — and what we don't

Tracked (aggregate counts only)

  • Keystroke count (not content)
  • Mouse movement distance (not position)
  • Left & right click counts
  • Scroll step totals
  • AI prompt counts (VS Code)
  • Session start / end timestamps

Never tracked

  • Keystroke content or sequences
  • Cursor X/Y coordinates
  • Window titles or app names
  • Screenshots or screen recordings
  • Network usage or visited URLs
  • Any personally identifiable data

All data is stored locally by default. Optional backend push is HTTPS-only and explicitly opt-in.

Integration

Push data to BLT Fresh (optional)

Wire up the tracker to a Cloudflare Worker in three lines of Dart. 

// lib/main.dart — initialise integration
final _integration = TrackerIntegration(
  apiUrl:     'https://your-worker.workers.dev',
  userId:     'user-123',
  projectId: 'project-456',
);
await _integration.startSession();

// on each 1-second ActivityService update:
_integration.sendActivityData(data);
